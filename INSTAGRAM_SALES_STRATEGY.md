## 1. Pricing Strategy

### Recommended Pricing Tiers

| Package | Price Range | What's Included |
|---------|-------------|-----------------|
| **Basic** | $500 - $800 | Static site (all pages: Home, Menu, About, Contact, Reservations), mobile responsive, contact form, Google Maps integration, basic SEO setup |
| **Standard** | $1,000 - $1,500 | Everything in Basic + functional cart system, reservation booking system, image optimization, basic analytics setup, social media integration |
| **Premium** | $1,800 - $2,500 | Everything in Standard + payment gateway integration (Stripe/PayPal), CRM integration (HubSpot/Mailchimp), advanced analytics, online ordering system, 1 month free hosting, priority support |

### Add-On Services (À La Carte)

| Add-On | Price | Description |
|--------|-------|-------------|
| Menu Management System | $200 - $400 | CMS for client to update menu items/prices independently |
| Multi-Language Support | $300 - $500 | Translation integration (English/Spanish, etc.) |
| Loyalty Program Integration | $250 - $400 | Points/rewards system for repeat customers |
| Online Ordering System | $400 - $700 | Full ordering flow with payment processing |
| Email Marketing Setup | $150 - $300 | Mailchimp/ConvertKit integration with signup forms |
| Review System Integration | $200 - $350 | Google/Yelp review widgets on site |
| Custom Photography Coordination | $300 - $600 | Arrange professional food photography (excluding photographer fees) |
| Advanced SEO Package | $250 - $500 | Keyword research, schema markup, Google Business optimization |

### Monthly Maintenance Packages (Optional Recurring Revenue)

| Tier | Monthly Price | Services Included |
|------|---------------|-------------------|
| **Basic Care** | $50/month | Hosting, security updates, monthly backup, 1 minor content update |
| **Standard Care** | $100/month | Everything in Basic + 2 content updates/month, basic analytics report, uptime monitoring |
| **Premium Care** | $150/month | Everything in Standard + unlimited minor updates, monthly SEO check, quarterly performance report, priority support |

### Pricing Factors to Consider

- **Client Location:** US/EU/Australia clients = higher pricing tier. Developing markets = adjust 20-40% lower
- **Restaurant Size:** Small local shop vs. upscale establishment vs. multi-location chain
- **Your Experience Level:** Beginner = lower end of range. Experienced portfolio = premium pricing
- **Project Timeline:** Rush projects (under 7 days) = add 25-50% surcharge
- **Always Include:** 3 months free support and revisions in base price

### Payment Structure

```
Deposit: 50% upfront before work begins
Final Payment: 50% upon project completion and approval

Optional Payment Plans:
- 3 monthly installments (for projects $1,500+)
- Accept: Stripe, PayPal, bank transfer, Wise
```

## 2. Client Requirements Checklist

### Before Building - Must Have From Client

#### Branding & Identity
- [ ] Business name (exact spelling/formatting)
- [ ] Logo files (SVG, PNG, or high-res AI/EPS)
- [ ] Brand colors (hex codes if available)
- [ ] Brand fonts (or permission to select appropriate fonts)
- [ ] Brand voice/tone preferences (casual, upscale, playful, etc.)

#### Content
- [ ] Complete menu with current prices
- [ ] High-quality food photography (10-20 minimum)
- [ ] Restaurant interior photos (3-5)
- [ ] Restaurant exterior photos (2-3)
- [ ] Team/chef photos and bios (optional)
- [ ] About story / restaurant history
- [ ] Tagline or slogan (if applicable)

#### Business Information
- [ ] Complete address (for Google Maps integration)
- [ ] Business hours (regular schedule)
- [ ] Special hours (holidays, events)
- [ ] Phone number
- [ ] Email address for contact form routing
- [ ] Social media handles (Instagram, Facebook, TikTok, Yelp, TripAdvisor)
- [ ] Delivery area/radius (if offering delivery)

#### Technical Requirements
- [ ] Domain name (do they have one? Do they need help purchasing?)
- [ ] Current hosting provider (if migrating from existing site)
- [ ] Existing website login credentials (if redesign)
- [ ] Google Business Profile access (optional, for integration)

### For Third-Party Integrations

#### Payment Processing (If Online Ordering)
- [ ] Payment gateway choice (Stripe recommended / PayPal / Square)
- [ ] Stripe API keys (Publishable Key + Secret Key)
  - *How to get:* stripe.com → Dashboard → Developers → API Keys
- [ ] PayPal Business account credentials (if using PayPal)
- [ ] Square API credentials (if using Square POS)
- [ ] Bank account info for payment deposits

#### Reservation System
- [ ] Current reservation platform (OpenTable / Resy / SevenRooms / Zomato)
- [ ] Platform API key or embed code
- [ ] Booking rules (party size limits, advance booking window, deposit requirements)
- [ ] Cancellation policy text
- [ ] Alternative: Use built-in form → email notifications (no API needed)

#### CRM & Email Marketing (Optional)
- [ ] CRM platform choice (HubSpot free / Mailchimp / ActiveCampaign)
- [ ] Account login or API key
- [ ] Email signup form preferences (popup, footer, dedicated page)
- [ ] Welcome email content (if applicable)
- [ ] Subscriber list (if migrating from existing system)

#### Online Ordering & Delivery
- [ ] Delivery platform integrations (UberEats / DoorDash / Grubhub links)
- [ ] In-house delivery parameters (fees, minimum order, zones)
- [ ] Pickup vs. delivery settings
- [ ] Order lead time requirements
- [ ] Tax rate(s)

#### POS Integration (If Applicable)
- [ ] POS system name (Toast / Square / Clover / Lightspeed)
- [ ] API access credentials
- [ ] Menu sync preferences
- [ ] Inventory management requirements

#### Analytics & Tracking
- [ ] Google Analytics account (create if doesn't exist)
- [ ] Google Tag Manager account (optional)
- [ ] Facebook Pixel ID (for ad retargeting)
- [ ] Google Search Console access

#### Hosting & Domain
- [ ] Domain registrar login (GoDaddy / Namecheap / Cloudflare)
  - *OR offer to purchase on their behalf*
- [ ] Hosting preference (Netlify / Vercel / SiteGround / managed by you)
- [ ] DNS access (for custom domain setup)
- [ ] SSL certificate (usually included with modern hosting)

## 3. Instagram DM Script Flow

### Step 1: Pre-Outreach Preparation (1-2 Weeks Before DM)

**Actions:**
- [ ] Follow their Instagram account
- [ ] Like 2-3 recent posts (genuine engagement)
- [ ] Leave 1 thoughtful comment on a post
- [ ] Study their brand, menu, vibe, customer base
- [ ] Check if they have a website (Google them)
- [ ] Note weaknesses in current online presence
- [ ] Build a quick demo/mockup inspired by their brand
- [ ] Identify the decision-maker (owner vs. social media manager)

**Research Checklist:**
- [ ] What type of cuisine? (Japanese/sushi in this case)
- [ ] Price point? (casual, mid-range, upscale)
- [ ] Location(s)? (single, multi-location)
- [ ] Current website? (none / outdated / needs improvement)
- [ ] Online ordering? (yes/no/through third-party)
- [ ] Do they take reservations?
- [ ] Recent reviews/ratings? (check Google, Yelp)
- [ ] Any recent news? (expansion, new chef, awards)

#### Scenario A: They Have NO Website


## 4. Demo Hosting Guide

### Free Hosting Options for Demos

#### Option 1: Netlify (RECOMMENDED - Easiest)

**Steps:**
1. Go to [netlify.com/drop](https://app.netlify.com/drop)
2. Drag and drop your project folder
3. Site goes live in seconds
4. Custom URL: `sushiwarriors.netlify.app`
5. Optional: Connect custom domain later

**Pros:** Free, fast, includes SSL, drag-and-drop, no coding needed
**Cons:** Netlify subdomain (unless you connect custom domain)

---

#### Option 2: Vercel

**Steps:**
1. Push code to GitHub
2. Connect GitHub to [vercel.com](https://vercel.com)
3. Import repository
4. Auto-deploys on every commit
5. Custom URL: `sushiwarriors.vercel.app`

**Pros:** Excellent for React/Next.js, auto-deploys, great performance
**Cons:** Requires GitHub knowledge

---

#### Option 3: GitHub Pages

**Steps:**
1. Push code to GitHub repository
2. Go to Settings → Pages
3. Select branch (usually `main`)
4. Site live at: `yourusername.github.io/sushiwarriors`

**Pros:** Free, integrated with GitHub, custom domain support
**Cons:** Slower updates, slightly more technical

---

#### Option 4: Render

**Steps:**
1. Connect GitHub to [render.com](https://render.com)
2. Create new static site
3. Auto-deploys from repository
4. Custom URL: `sushiwarriors.onrender.com`

**Pros:** Free SSL, auto-deploys, professional
**Cons:** Slightly slower initial load

---

### Pro Tips for Demo Hosting

✅ **Use Netlify for quickest results** - literally drag and drop
✅ **Create a branded subdomain** - `demo.youragency.com/restaurant-name` (requires custom domain)
✅ **Password-protect previews** - Netlify allows this on paid plans
✅ **Take screenshots** - Always have static images ready to send in DMs
✅ **Record a Loom video** - 60-90 second walkthrough increases conversion massively
✅ **Mobile test** - Ensure demo works perfectly on phones before sending
✅ **Add a watermark** - "Demo by [Your Name]" in footer to prevent theft
✅ **Track views** - Use Google Analytics on demo to see if they're checking it

## 6. Pro Tips for Instagram Sales

### Before You Pitch

✅ **Build your own Instagram presence first**
   - Post your work, behind-the-scenes, client results
   - Have a professional bio with "I build restaurant websites 🍣🌐"
   - Link to your portfolio in bio

✅ **Warm up before outreach**
   - Follow target restaurant 1-2 weeks before pitching
   - Genuinely engage with their content (likes, comments)
   - Share their posts to your stories (tag them)
   - Become a familiar name before the pitch

✅ **Research thoroughly**
   - Study their menu, vibe, target customer
   - Check competitors' websites
   - Note what's missing in their online presence
   - Build a custom demo BEFORE reaching out

### Crafting Your Pitch

✅ **Personalize every single message**
   - Use their restaurant name
   - Mention specific dishes/posts
   - Reference something unique about them
   - NEVER send copy-paste spam

✅ **Lead with value, not selling**
   - "I built something for you" vs. "I can build something for you"

   - Give before asking

✅ **Keep it conversational**
   - Short messages (2-3 short paragraphs max)

✅ **Use social proof**

   - Have portfolio/testimonials ready to share
   - Show results, not just pretty designs

### Follow-Up Strategy

✅ **Follow up 3-5 times minimum**

## 7. Red Flags to Avoid

### Client Red Flags 🚩

❌ **"Can you do it for free/exposure?"**
   - Exposure doesn't pay bills
   - They don't value your work
   - Walk away politely: "I appreciate the offer, but I run this as my business"

❌ **No clear decision-maker**
   - "I need to check with my partner/manager/investor" (repeatedly)
   - You're talking to the wrong person
   - Ask upfront: "Who makes the final call on this?"

❌ **Unrealistic expectations**
   - "I want Amazon but my budget is $200"
   - "Can you do it in 2 days?"
   - Set boundaries early

❌ **Wants unlimited revisions without scope**
   - Define revision rounds in contract (2-3 rounds)
   - "Unlimited tweaks" = endless nightmare
   - Charge extra for scope creep

❌ **Expects 24/7 availability**
   - DMs at midnight expecting instant replies
   - Set communication boundaries
   - "I respond within 24 hours on business days"

❌ **No budget transparency**
   - Refuses to discuss budget range
   - "How cheap can you go?"
   - Be upfront about your rates

❌ **Already had a bad experience with another developer**
   - They may project that trauma onto you
   - "Last guy screwed us over"
   - Proceed with extra-clear contracts and boundaries

❌ **Asks for your source files before paying**
   - Files transfer after final payment
   - Standard industry practice
   - Don't budge on this

❌ **Wants you to work on spec**
   - "Build it first, I'll pay if I like it"
   - No. Deposit first, always.
   - Your time has value

❌ **Multi-location enterprise but wants "simple site for $300"**
   - They don't understand the scope
   - Educate them or walk away
   - Enterprise clients should pay enterprise rates

### Project Red Flags 🚩

❌ **Constant scope changes mid-project**
   - "Can we also add delivery tracking, a blog, and a merch store?"
   - Refer to contract, charge for additions
   - "Happy to add that! Here's the additional cost..."

❌ **Client is unreachable**
   - Doesn't send content/photos on time
   - Disappears during review phases
   - Set deadlines: "Content needed by [date] to stay on schedule"

❌ **Micromanaging every pixel**
   - Trust your expertise
   - "Move this 2px left" × 50 times
   - Set boundaries: "Let's consolidate feedback into 1-2 revision rounds"

❌ **Third-party involvement mid-project**
   - "My cousin's friend wants to review it"
   - "Our marketing agency needs to approve"
   - Clarify stakeholders upfront
